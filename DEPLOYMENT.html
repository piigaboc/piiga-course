PiigaCourse — Deployment Guide

Single-user online-course tracker · React 18 + Vite + TS frontend · FastAPI + SQLAlchemy 2.0 async backend · PostgreSQL

Auth is password + TOTP MFA (JWT). Features include a courses list and a study calendar with a GitHub-style green heatmap, in a warm "stone" theme. This guide covers deploying to Neon (database) → Render (backend) → Vercel (frontend), plus every gotcha hit along the way.

#Architecture

Vercel ──HTTPS──▶ Render ──asyncpg/SSL──▶ Neon (static Vite SPA) (FastAPI web service) (Postgres) VITE_API_BASE_URL ──▶ https://<render>/api CORS_ORIGINS ──▶ https://<vercel> DATABASE_URL ──▶ Neon connection string

Each hop is wired by one environment variable: the frontend's VITE_API_BASE_URL points at the Render URL + /api; Render's CORS_ORIGINS must list the Vercel URL; Render's DATABASE_URL points at Neon.

1Neon — database (do this first)

  1. Create a Neon project. Pick a region close to your Render region.
  2. Copy the connection string from the Neon dashboard. It looks like: 
    postgresql://user:pass@ep-xxx.us-east-2.aws.neon.tech/neondb?sslmode=require&channel_binding=require
  3. Paste that raw URL straight into Render's DATABASE_URL — no editing.
No manual URL surgery needed. The backend auto-normalizes the connection string: it rewrites the scheme to postgresql+asyncpg://, strips the libpq-only params asyncpg rejects (sslmode, channel_binding, …), and enables SSL via connect_args for remote hosts. Local localhost URLs get no SSL.

Keep SECRET_KEY stable. It signs JWT login tokens. If you change it after going live, every existing session is invalidated and users are forced to log in again.

The single user account is created automatically on startup from BOOTSTRAP_* (idempotent). Check the Render Logs for Bootstrap: created single user … — if you see Bootstrap SKIPPED, the bootstrap vars are missing or placeholder.

3Vercel — frontend (Vite SPA)

  1. New Project → import the repo → Root Directory: frontend.
  2. Framework preset: Vite (Build npm run build, Output dist).
  3. Add env var: VITE_API_BASE_URL = https://<service>.onrender.com/api
  4. Deploy. Then set Render's CORS_ORIGINS to the exact Vercel URL (no trailing slash) and redeploy the backend.
Vite bakes env vars at build time. After changing VITE_API_BASE_URL you must Redeploy — it won't take effect on a live build. Only VITE_-prefixed vars are exposed to the frontend.

Order of operations: deploy backend first to get its URL → deploy frontend with that URL → set backend CORS_ORIGINS to the Vercel URL → redeploy backend.

4Database migrations (Alembic)

Run from the backend directory (where alembic.ini lives). On Render, set it as the Pre-Deploy Command or run once in the Shell tab.

CommandResult
alembic upgrade head✅ correct (CLI)

Locally, make sure DATABASE_URL and SECRET_KEY are set (or in backend\.env) first:

cd D:\claude\piigacourse\backend

5Local development (quick reference)

# 1. Postgres
docker compose up -d

# 2. Backend (PowerShell)
cd backend
py -3.12 -m venv .venv
.\.venv\Scripts\Activate.ps1
pip install -r requirements.txt
Copy-Item .env.example .env        # then edit values
alembic upgrade head
python -m app.bootstrap            # seed the single user
uvicorn app.main:app --reload --port 8000   # http://localhost:8000/api/docs

# 3. Frontend
cd ..\frontend
npm install
npm run dev                        # http://localhost:5173

In local dev the Vite dev server proxies /api → localhost:8000, so VITE_API_BASE_URL can stay /api.

6Troubleshooting

SymptomCauseFix
Field required [type=missing] … SECRET_KEY SECRET_KEY not set on Render. Add it (≥32 chars) in Environment → redeploy.
ModuleNotFoundError: No module named 'psycopg' Service running OLD code, or a plain postgresql:// URL on a build before the normalizer. Redeploy the latest commit. Current code auto-converts to +asyncpg — just paste Neon's raw URL.
can't find '__main__' module in '…/alembic' Ran python alembic upgrade head (executes the folder). Use alembic upgrade head or python -m alembic upgrade head.
Root URL returns {"detail":"Not Found"} Not an error — there's no route at /. Hit /api/health; it confirms the app is up.
SECRET_KEY rejected (ValueError) Value is empty, a known placeholder, or under 32 chars. Use a real random ≥32-char value.
Log shows Bootstrap SKIPPED BOOTSTRAP_EMAIL/BOOTSTRAP_PASSWORD unset or placeholder. Set strong values → redeploy → account is created.
CORS errors in browser console Render CORS_ORIGINS doesn't match the Vercel origin exactly. Set it to the exact scheme + host (no trailing slash) → redeploy backend.
First request after idle takes ~50s Render free tier cold start. Expected on the free plan; upgrade for always-on.
Logins suddenly invalid SECRET_KEY was changed. Keep it stable; restore the original value.
